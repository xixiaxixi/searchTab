# 筛选语法说明

这个扩展支持强大的筛选功能，允许你使用多种方式筛选收藏夹和历史记录。

## 语法规则

### 1. 基本文本匹配（默认）
直接输入关键词，会在**标题和URL**中搜索（不区分大小写）。

**示例：**
```
github
```
匹配标题或URL中包含"github"的所有项目。

---

### 2. 仅匹配标题
使用 `title:` 前缀，只在标题中搜索。

**示例：**
```
title:新闻
```
只匹配标题中包含"新闻"的项目，URL中的"新闻"会被忽略。

---

### 3. 仅匹配URL
使用 `url:` 前缀，只在URL中搜索。

**示例：**
```
url:.com
```
只匹配URL中包含".com"的项目。

---

### 4. 正则表达式匹配
使用 `/pattern/` 格式，在标题和URL中使用正则表达式搜索。

**示例：**
```
/^https:\/\/github\.com/
```
匹配所有以 `https://github.com` 开头的URL。

```
/新闻|资讯/
```
匹配标题或URL中包含"新闻"或"资讯"的项目。

---

### 5. 标题正则匹配
结合字段前缀和正则表达式：`title:/pattern/`

**示例：**
```
title:/^GitHub/
```
只匹配标题以"GitHub"开头的项目。

---

### 6. URL正则匹配
结合字段前缀和正则表达式：`url:/pattern/`

**示例：**
```
url:/\.com$/
```
只匹配URL以".com"结尾的项目。

```
url:/^https:\/\/(www\.)?(github|gitlab)\.com/
```
只匹配来自GitHub或GitLab的URL。

---

## 实用示例

### 示例1: 筛选特定域名
```
url:/github\.com/
```
匹配所有GitHub的页面。

### 示例2: 筛选特定协议
```
url:/^https/
```
只显示HTTPS的网址。

### 示例3: 排除特定内容（使用负向预查）
```
/^(?!.*广告)/
```
排除标题或URL中包含"广告"的项目。

### 示例4: 多个关键词（OR）
```
/文档|教程|指南/
```
匹配包含"文档"、"教程"或"指南"任一关键词的项目。

### 示例5: 匹配特定文件类型
```
url:/\.(pdf|doc|docx)$/
```
只显示PDF或Word文档的链接。

### 示例6: 中文标题
```
title:/[\u4e00-\u9fa5]/
```
只显示标题包含中文字符的项目。

---

## 正则表达式修饰符

默认情况下，正则表达式是**不区分大小写**的（i标志）。你可以添加其他修饰符：

```
/pattern/gi    - 全局匹配，不区分大小写
/pattern/g     - 全局匹配，区分大小写
/pattern/      - 默认（不区分大小写）
```

---

## 错误处理

如果正则表达式语法错误，系统会：
1. 在保存时提示错误信息
2. 自动回退到普通文本搜索（如果在运行时出错）

---

## 提示

- 正则表达式需要用 `/` 包裹，如 `/pattern/`
- 字段前缀（title:、url:）不区分大小写
- 普通文本搜索总是不区分大小写
- 可以先在正则表达式测试网站（如 regex101.com）测试你的正则表达式

---

## 快速参考

| 语法 | 说明 | 示例 |
|------|------|------|
| `keyword` | 全文本搜索 | `github` |
| `title:keyword` | 仅标题文本搜索 | `title:教程` |
| `url:keyword` | 仅URL文本搜索 | `url:.com` |
| `/regex/` | 全文正则搜索 | `/^https/` |
| `title:/regex/` | 仅标题正则搜索 | `title:/^新闻/` |
| `url:/regex/` | 仅URL正则搜索 | `url:/github\.com/` |
